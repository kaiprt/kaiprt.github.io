Dynamic and Static Languages both have there pros and cons my job today is to teach you what I have learned so far in the debate between Dynamic typed languages and Static typed languages.

Static Typed Languages

Static type-checking is the process of verifying the type safety of a program based on analysis of a program's text (source code). If a program passes a static type-checker, then the program is guaranteed to satisfy some set of type-safety properties for all possible inputs. (Wikipedia) This is beneficial because it is easier to detect programming mistakes, more opportunities for compiler optimizatons that can lead one to believe a well typed program cannot go wrong. Some examples of static languages include assembly languages, C, C++, Objective-C, and Java.

A language is dynamically typed if the type of a variable is interpreted at runtime. This allows programmers to work more quickly because the type does not have to be specified everytime. Also the code appears to be much "softer". A dynmaic programmer may find static languages to be "rigid". The downfall of a dynamic language is the fact that you cannot find errors as quickly as its competitor. I believe this is countered by creating assert statements, drivercode, and using test-driven development. Examples of Dynamic languages include Perl, Ruby, JavaScript and Python.

Benefits from dynmaic programming:

  • Dynamic languages are easier to read because you write less code.
  • Programming is faster
